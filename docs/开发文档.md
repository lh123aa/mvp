# 开发文档

## 概述

本文档详细介绍 Demand Miner 项目的技术架构、模块设计和开发规范。

---

## 技术架构

### iFlow CLI Skills 架构

Demand Miner 基于 iFlow CLI 的 Skills 框架构建，采用标准化的技能配置和执行模式。

```
┌─────────────────────────────────────────┐
│           iFlow CLI Engine              │
├─────────────────────────────────────────┤
│  skill.yaml (触发配置)                  │
│  SKILL.md (技能文档/SOP)               │
├─────────────────────────────────────────┤
│  skills/__init__.py (入口)              │
├─────────────────────────────────────────┤
│  scripts/ (业务逻辑)                    │
│  resources/ (数据资源)                 │
└─────────────────────────────────────────┘
```

## 模块设计

### 1. 技能入口 (skills/__init__.py)

**职责**：作为 Skills 包的入口点，加载和导出核心功能

**功能**：
- 读取 SKILL.md 文档内容
- 导出技能元数据（名称、版本、描述）
- 提供模块导入接口

```python
# 核心导出
__all__ = ['SKILL_CONTENT', 'get_skill_info']
```

### 2. 辅助脚本 (scripts/)

#### 2.1 constraint_filter.py

**职责**：约束过滤逻辑，根据用户预算、周期和团队人数过滤方案

**函数**：
```python
def filter_by_budget(solutions: list, budget: str) -> list
def filter_by_timeline(solutions: list, timeline: str) -> list
def filter_solutions(solutions: list, constraints: dict) -> list
```

**约束规则**：
| 预算 | 过滤规则 |
|------|----------|
| 0成本 | 过滤所有付费方案 |
| 5000元内 | 过滤成本>5000的方案 |
| 5000-20000元 | 过滤成本>20000的方案 |

| 周期 | 过滤规则 |
|------|----------|
| 1个月 | 过滤周期>1个月的方案 |
| 3个月 | 过滤周期>3个月的方案 |
| 6个月 | 过滤周期>6个月的方案 |

| 团队人数 | 过滤规则 |
|----------|----------|
| 1人 | 过滤需要多人的方案 |
| 1-3人 | 过滤需要>3人的方案 |
| 5人以上 | 无特殊限制 |

#### 2.2 data_loader.py

**职责**：加载行业数据和问题配置

**函数**：
```python
def load_industries() -> dict
def load_questions() -> dict
def get_industry_data(industry: str) -> dict
```

#### 2.3 review.py

**职责**：对话复盘逻辑

**函数**：
```python
def create_review_from_session(session_data: dict) -> str
def save_review_to_file(review_content: str, file_path: str) -> bool
```

## SKILL.md 规范

SKILL.md 是技能的核心文档，遵循以下结构：

### 必需章节

1. **元数据**：版本、作者、触发关键词
2. **角色定义**：AI 角色和能力
3. **核心能力**：主要功能描述
4. **输出格式**：标准化输出模板
5. **响应规则**：交互约束

### 触发关键词配置

在 skill.yaml 中配置：
```yaml
triggers:
  - 产品分析
  - 项目分析
  - 商业分析
  - 商业闭环
  - 需求挖掘
  - 商业模式
  - 创业分析
  - 副业规划
  - 市场分析
  - 用户调研
  - 竞品分析
  - 复盘
```

## 开发规范

### 代码风格

- 使用 Python 3.x 语法
- 函数命名使用 snake_case
- 添加必要的类型注解
- 保持函数简洁单一职责

### 文件命名

- Python 模块：snake_case.py
- 配置文件：kebab-case.yaml / camelCase.json
- 文档：kebab-case.md

### Git 提交规范

```
feat: 新功能
fix: 修复
refactor: 重构
docs: 文档更新
chore: 其他更新
```

## 对话示例

### 示例1：副业试水场景

**用户输入**：`副业规划`

**对话流程**：

```
助手：你本次想要拆解分析的核心目标，是具体的职业、某类商业赛道，还是一款特定的产品？请同时告知该目标的具体名称或赛道方向。

用户：我想做短视频带货

助手：你本次做需求挖掘与最小闭环设计的核心目标是什么？
A. 副业试水落地
B. 创业立项分析
C. 行业研究学习
D. 职业转型规划

用户：A

助手：你对这个目标领域，目前的认知程度符合以下哪一种？
A. 完全陌生：仅听过名字，完全不了解行业规则、玩法
B. 有基础认知：了解行业基础定义、主流玩法，有少量相关了解
C. 资深从业者：深耕该领域，有丰富的实操经验与行业认知

用户：B

助手：你希望这个最小商业闭环的落地周期与可投入的成本预算上限大概是怎样的？
A. 0成本，1个月内落地
B. 低预算（5000元内），3个月内落地
C. 中预算（5000-20000元），6个月内落地

用户：B

助手：你的团队规模是？
A. 1人
B. 1-3人
C. 5人以上

用户：A
```

**输出**：
```

- 核心目的：副业试水落地
- 认知层级：有基础认知
- 预算约束：5000元内
- 周期约束：3个月
- 团队人数：1人
- 服务模式：引导教练

[继续输出完整分析...]
```

## 扩展开发

### 添加新行业

1. 在 `resources/industries.json` 中添加行业数据
2. 遵循现有结构：用户群体、方案、痛点、验证指标

### 添加新功能

1. 更新 `SKILL.md` 添加功能描述
2. 在 `scripts/` 中实现对应逻辑
3. 更新 `skill.yaml` 触发词（如需要）

## 常见问题

### 基础问题

**Q: 如何修改触发关键词？**
A: 编辑 `skill.yaml` 中的 `trigger_keywords` 列表

**Q: 如何添加新的分析模式？**
A: 在 `SKILL.md` 中添加模式描述，并在 `skills/__init__.py` 中实现

**Q: 数据文件格式有什么要求？**
A: 使用标准 JSON 格式，确保 UTF-8 编码

### 功能问题

**Q: 如何添加新的约束类型？**
A: 
1. 在 `SKILL.md` 中添加问题选项
2. 在 `constraint_filter.py` 中添加解析逻辑
3. 在 `scripts/review.py` 中添加评估逻辑

**Q: 复盘功能如何触发？**
A: 用户输入"复盘"或"结束对话"时自动触发

**Q: 如何扩展行业数据？**
A: 在 `resources/industries.json` 中添加新行业，遵循现有 JSON 结构

### 技术问题

**Q: 如何运行单元测试？**
A: 
```bash
python tests/test_constraint_filter.py
python tests/test_data_loader.py
```

**Q: 如何调试脚本？**
A: 在脚本中添加 print 语句，直接运行脚本文件
